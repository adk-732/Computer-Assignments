Keys in DBMS

What are Keys in DBMS?

In a Database Management System (DBMS), a key is a single or combination of multiple columns in a table that uniquely identifies a record in the table.

Types of Keys

  • Primary Key (PK)

    A primary key is a column or set of columns that uniquely identifies each record in a table. It is used to enforce data integrity and ensure that each record is unique. A primary key cannot be null, and it must be unique for each record.

  • Unique Key (UK)

    A unique key is a column or set of columns that uniquely identifies each record in a table, but can be null. It is used to ensure that each record has a unique value for the specified column(s), but it does not enforce data integrity like a primary key.

  • Foreign Key (FK)

    A foreign key is a column or set of columns that references the primary key of another table. It is used to establish relationships between tables and ensure data consistency.

  • Composite Key

    A composite key is a primary key that consists of multiple columns. It is used when a single column is not enough to uniquely identify each record in a table.

  • Super Key

    A super key is a set of columns that uniquely identifies each record in a table, but may not be minimal. It is a superset of a candidate key.

  • Candidate Key

    A candidate key is a minimal super key that uniquely identifies each record in a table. It is a set of columns that can be used as a primary key.

Importance of Keys

Keys play a crucial role in maintaining data integrity and consistency in a database. They help to:

  • Uniquely identify each record in a table
  • Prevent data duplication
  • Establish relationships between tables
  • Improve data retrieval and query performance
